# 更新說明 — 2026-06-12（銷售訂單查詢：補欄位、部分單號查詢、複製對位）

> 影響範圍：美編天地 →「銷售訂單查詢」單一頁面，其他卡片與功能不受影響。

## 給所有同仁：這次改了什麼

### 1. 打部分單號就能查
以前要輸入完整訂單號（例：`SO260603001`）才查得到。現在**輸入其中一段就可以**，例如打 `0603`，所有單號含 0603 的訂單都會列出來（大小寫不分）。

- 一次最多列 500 筆；若太多會提示「請輸入更完整的單號」，多打幾個字就好。

### 2. 查詢結果補上出單表需要的欄位
- 新增「**出貨備註**」欄（ERP 的備註2）。
- 原本的「商品備註」欄改名為「**PACKING**」——它顯示的本來就是 ERP 包裝欄的內容，現在名稱跟出單表一致。
- 「前置單號」欄就是出單表的**打樣單號**（名稱維持不變）。

### 3. 「複製 N 筆」貼到出單表不再錯位
之前貼上後從 PACKING 欄開始會錯一格（數量跑進 PACKING、交期跑進數量、交付日期變空白）。現在複製格式已對齊出單表：

| 出單表欄位 | 貼上內容 |
|---|---|
| E 打樣單號 | ERP 前置單號 |
| H 客戶/供應商名 | 客戶名稱 |
| J 承辦人 | 業務 |
| L 品項編碼 | 料號 |
| M 品名/規格 | 品名 |
| N 出貨備註 | ERP 備註2 |
| O PACKING | ERP 包裝欄 |
| P 數量 / Q 交付日期 | 數量 / 交期 |
| R 盤數 | **不會被覆蓋**（維持人工填寫） |

## 你需要做什麼
- 不需要任何設定，照常使用即可。
- 提醒：查詢結果仍以「同步銷售訂單」的最新同步為準，資料怪怪的先按同步。

有問題請聯絡系統管理員。

## 附錄（技術摘要，給維護者）

僅改動 `app/design-studio/so-query/page.tsx`（前端單檔），不動同步邏輯、資料庫 schema 與其他頁面：

- **模糊查詢**：`handleQuery` 由 `.in('project_id', ids)` 改為 `.ilike('project_id', '%{input}%')`，新增 `.limit(QUERY_LIMIT)`（500）；達上限時 footer 顯示提示。`erp_so_lines` 全表約 5,400 筆，ilike 全掃描成本可忽略。
- **表格**：新增「出貨備註」欄渲染 `remark2`（欄位原本就在 select 清單，僅未顯示）；「商品備註」表頭改名「PACKING」（仍渲染 `packing`）；`COLS` 12 → 13。
- **複製對位**：`buildExcelRow` 改為 17 格對齊出單表 A~Q——N=`remark2`、O=`packing`、P=數量、Q=交付日期；刻意**不輸出第 18 格（R 盤數）**，避免貼上時清掉人工填寫內容。
- 欄位語意（使用者 2026-06-12 確認）：出貨備註=ERP 備註2（REMARK2）、PACKING=ERP 包裝欄（PACKING）、打樣單號=TPN_PART_NO（畫面名稱維持「前置單號」）。
